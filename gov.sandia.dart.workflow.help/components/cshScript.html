NGW Component Documentation - cshScript

cshScript

Description

Executes a csh script. In its simplest usage, the script can read data from stdin and print results on stdout. Additional inputs, outputs, and properties can be defined to create more elaborate behaviors. The csh interpreter is located using the environment variable NGW_INTERP_CSH.

Properties

script - the text of the script to execute.

write properties to a property file -- if 'true', then the node will write a Java properties file containing the node properties and the port values and save it in the node's working directory before executing the script.

Input Ports

stdin - data on this input become the standard input of the process.

script - path to a script to execute. Will override the 'script' property.

Output Ports

stdout - data on this output comes from the standard output of the process.

stderr - data on this output comes from the standard error of the process.

exitStatus - exit status of the process. If this is not connected and the script exits with nonzero status, this node will abort the workflow with a diagnostic.

Notes

The values of all node properties and data from all input ports, along with the values of global parameters and many standard workflow parameters, will be available to the script in variables of the same name. Periods ('.') and spaces in those names are replaced with underscores.
