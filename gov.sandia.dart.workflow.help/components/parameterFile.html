NGW Component Documentation - parameterFile

parameterFile

Description

Reads parameters from a file

Properties

fileName - the file to be read. The format should be that of a simple Java properties file: each line should be of the form "key=value". Octothorpes (#) introduce comment lines. No escaping of special characters or line continuation is performed.

Input Ports

None

Output Ports

f - a map containing all the parameter mappings from the file.

Notes

The parameterFile node will actually be executed before the rest of the workflow runs, and the parameters will be injected into the workflow engine at that time. If you connect the output of this node the map will be available for further processing, but an unconnected parameterFile node is the usual case.

If the given filename is not an absolute path, it is taken to be relative to the workflow home directory.
