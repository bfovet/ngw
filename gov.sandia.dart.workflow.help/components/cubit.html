NGW Component Documentation - cubit

cubit

Description

Runs the CUBIT meshing and geometry program. You can supply a journal file, a Python script, or both; this component will run them in that order.

To attach any additional input files, create an property type "local_file" and set the value to the local path to the file. The file will be part of the job and will be soft linked into the current directory when Cubit is invoked.

To access any generated files, create an output port of type "output_file," and set the metadata field to the name of the file. The absolute path to the file will then be provided on that port.

Properties

journalFile - A local journal file path to execute. Optional - if not set, value is read from the port of the same name.

pythonFile - A local Python file path to execute inside cubit. Optional - if not set, value is read from the port of the same name.

Input Ports

journalFile - Expects the path to a journal file to execute. Optional, but if connected, overrides the property of the same name.

pythonFile - Expects the path to a Python file to execute. Optional, but if connected, overrides the property of the same name.

Output Ports

logFile - Cubit-s stdout.

Notes

It is an error to specify neither a journal file nor a python script.
