NGW Component Documentation - sierra

sierra

Description

Submits a job to the queue manager on the local machine using a set of scripts (which can be modified by the user) and then waits for it to complete. Comes configured to submit a sierra job to slurm on the Sandia HPC clusters.

Properties

inputFile - An input file to execute -- by default, a Sierra input deck.

account - a WCID number to use.

sierra_code - the name of the sierra code to execute (adagio, fuego, etc.)

num.nodes - the number of compute nodes to request.

num.processors - the total number of processors to request.

job.hours - the number of hours of queue time to request.

job.minutes - the number of minutes of queue time to request.

submitScript - path to a bash script that submits a job to queue.

executeScript - path to a bash script will be executed in the queue.

checkjobScript - path to a bash script that checks whether the job has completed.

checkjobFrequency - number of seconds to wait before running the checkjob script (default is 15s).

statusScript - path to a bash script that will be run after job completion to learn if the job succeeded.

Input Ports

inputFile - A sierra input file. Optional, but if connected, overrides the property of the same name. It a file name or relative path, will be interpreted relative to the home directory

Output Ports

logFile - The absolute path to the Sierra output log.

Notes

It is an error to not specify an inputFile.

In the settings editor for this component, there is a file icon button next to the submitScript, executeScript, checkjobScript and statusScript properties. If you press it, the workflow editor will save a copy of the default script of that type to your workflow home directory, and insert the name of the saved file into the text field. You can then edit the script to suit.

This is a prototype capability.
