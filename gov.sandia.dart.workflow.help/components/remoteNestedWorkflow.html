NGW Component Documentation - remoteNestedWorkflow

remoteNestedWorkflow

Description

Represents a "sub-workflow," stored in a different IWF file from the main workflow, which will be executed remotely. The parameters and responses of the sub-workflow can be connected to input and output ports, respectively, of this node by pressing the context button. The remote workflow is executed synchronously on the remote host.

This node is powerful but not yet as convenient to use as we'd like. A future version will include a custom editor configured with known machines and their properties.

Properties

fileName - the name of the sub-workflow IWF file.

hostname - the name of the remote host. It is assumed that you can log into that host using the active Kerberos ticket.

remotePath - the absolute path on the remote machine in which the workflow should be executed. Like all workflow node properties, you can use parameters to define this value.

wflib (optional) - if present, specifies the location of a workflow library directory on the remote machine. If not present the standard search path is used.

Input Ports

This node has no standard input ports, but parameters of the sub-workflow can be connected to input ports if desired.

Output Ports

stdout - the standard output of the sub-workflow process.

stderr - the standard error of the sub-workflow process.

Additional output ports can be connected to the responses of the sub-workflow if desired. In addition, note that when the remote workflow completes its workflow.engine.log is retrieved, and the "Grab Output File" tool can be used to provide that file via a port.

Notes

This node logs copious progress info to both workflow.engine.log and the workflow stdout.
