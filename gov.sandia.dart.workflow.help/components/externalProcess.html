NGW Component Documentation - externalProcess

externalProcess

Description

Executes a command line. In its simplest usage, the command line can read data from stdin and print results on stdout. Additional inputs, outputs, and properties can be defined to create more elaborate behaviors.

Properties

command - the command line to execute. Each separate command line token should be enclosed in square brackets.

write properties to a property file -- if 'true', then the node will write a Java properties file containing the node properties and the port values and save it in the node's working directory before executing the script.

Input Ports

stdin - data on this input become the standard input of the process.

Output Ports

stdout - data on this output comes from the standard output of the process.

stderr - data on this output comes from the standard error of the process.

exitStatus - exit status of the process. If this is not connected and the script exits with nonzero status, this node will abort the workflow with a diagnostic.

Notes

Global and built-in workflow parameters can be used via variable interpolation to build the command line.

Connections can be used to save files to a node's workdir, and the "Grab Output File" tool can be used to collect results.
